Проект для демонстрации работы плагинов 

Unity_Plugin_Wrapper_Addressables_Manager_Public - https://github.com/ximik-xim/Unity_Plugin_Wrapper_Addressables_Manager_Public

Unity_Plugin_Scene_Level_Manager_Public - https://github.com/ximik-xim/Unity_Plugin_Scene_Level_Manager_Public

Концепция следующая, плагин Addressables используется для обновления данных в игре с сервера(загрузка уровней), это необходимо в случае если надо добавить контента и при этом нет не каких изменении в коде.

Но что бы игрок мог скачать игру и поиграть в неё, даже если сервер не отвечает, то в игру зашиты сразу несколько резервныйх уровней. Это приводит к дублированию ресурсов, т.к в обновлениях с сервера эти же ресурсы будут. 

Зато игрок сможет в любом случае опробывать игру.

---------------------------------------------------------------------------------------------------------

Репозиторий необходимо клонировать в папку с максимально коротким путем. Пр C:/Test

Это связано с ограничением длины пути 260 символов, если путь будет длиннее, то при клонировании репозитория получите ошибку error: unable to create file ... Filename too long

В ином случае необходимо включить поддержку более длинных путей как для Git, так и на самой Windows.
